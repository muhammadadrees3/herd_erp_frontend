1.1 THEME SYSTEM
----------------
The app has a full DARK / LIGHT theme toggle, managed globally via ThemeContext.
Every component reads `isDark` and switches all colors accordingly.

  Dark Mode Background:   bg-neutral-950 / bg-neutral-900 (near-black)
  Light Mode Background:  bg-white / bg-neutral-50 / bg-neutral-100
  Root page uses a gradient:
    Dark:  bg-gradient-to-br from-gray-950 via-gray-900 to-gray-950
    Light: bg-gradient-to-br from-gray-50 via-white to-gray-50

1.2 TYPOGRAPHY
--------------
Two Google Fonts are used throughout every component:

  PRIMARY HEADING FONT:  Space Grotesk
    Weights used: 300, 500, 700
    Used for: All major headings (h1, h2, h3), hero titles, card titles
    Style: UPPERCASE, tight tracking (tracking-tighter), bold, condensed leading

  BODY / UI FONT:  Inter
    Weights used: 300, 400, 600
    Used for: Navigation labels, paragraphs, descriptions, form labels

  MONOSPACE (accent):  Geist Mono (from Geist via Next.js) + Inter mono classes
    Used for: Technical labels, IDs, code-style hints, stat sub-labels
    Example: "FARM-A102", "LOG-001", "PRICE // MONTHLY", "// LIVESTOCK_PLATFORM"

  Typography sizes for headings:
    Hero H1:      text-5xl → text-7xl → text-8xl → text-9xl (very large)
    Section H2:   text-4xl → text-6xl → text-8xl
    Card H3:      text-xl → text-2xl → text-3xl
    Labels:       text-[9px] → text-[11px] (very small uppercase mono)

1.3 COLOR PALETTE
-----------------
  PRIMARY ACCENT:    Green (green-400, green-500, green-600, green-700)
                     Used for: CTAs, active states, icons, borders, gradients
  TEXT PRIMARY:      White (dark) / Black (light)
  TEXT SECONDARY:    neutral-400 (dark) / neutral-600 (light)
  TEXT MUTED:        neutral-500 / neutral-600
  BORDERS DARK:      white/5  white/10  white/20  green-500/20  green-500/30
  BORDERS LIGHT:     neutral-200  neutral-300  green-200  green-500/30
  ERROR:             Red-500 (used in bg-red-500/10 with border-red-500/30)
  SUCCESS:           Green-500 (bg-green-500/10 with border-green-500/30)

1.4 OVERALL DESIGN AESTHETIC
-----------------------------
The design follows a "Technical Brutalist" or "Industrial Dashboard" aesthetic:
  - Sharp rectangular cards (no round corners except a few selective ones)
  - Thin technical border lines, including corner accent markers (2px corner borders)
  - Grid background overlays (subtle linear gradients forming grid patterns)
  - Technical monospace labels like "LOG-001", "FARM-A102", "// REAL-TIME"
  - Glowing green accents (green soft glow shadows: shadow-[0_0_30px_rgba(34,197,94,0.3)])
  - Everything in UPPERCASE for labels and buttons
  - Scan-line animations (a green line sweeping downward like a radar/scan effect)

1.5 BACKGROUND DECORATION (used on most pages)
-----------------------------------------------
  Most sections use these layered background elements:
  1. A subtle grid: bg-[linear-gradient(to_right,#80808012_1px...)] bg-[size:24px_24px]
  2. A large radial soft glow blob (500px, blur-[120px]) in green-900/10 or green-500/5
  - These are `pointer-events-none` overlays, purely decorative

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   - Header becomes opaque and shrinks after 20px scroll
   - Hero text + background have parallax scroll effects

2. MICRO-ANIMATIONS:
   - Hover underlines on nav links (w-0 → w-full)
   - Button arrows translate-x on hover
   - Cards lift on hover (-translate-y-4) in pricing section
   - Green glow intensifies on hover across many interactive elements
   - Icons scale(1.1) on active/hover
   - Avatar scales on hover in navbar

3. INFINITE LOOPS:
   - Image ticker scrolls left forever (hero bottom)
   - Testimonial columns scroll up/down forever
   - Scan lines animate top-to-bottom forever
   - Footer progress bar slider animates forever
   - Sidebar accent line slides back and forth

4. THEME SWITCHING:
   - Instant toggle, all colors defined conditionally
   - Dark: near-black neutrals, green accents, subtle white borders
   - Light: white/neutral backgrounds, stronger borders, same green accents

5. LOADING / FEEDBACK STATES:
   - Login/Signup submit button shows "Logging In..." / "Creating Account..."
   - Error: red box with text; Success: green box with text
   - Profile photo upload disables button and shows "Uploading..."

6. MODAL / OVERLAY:
   - Video modal opens with spring animation, closes on ESC or backdrop click
   - Mobile menu slides in from right (spring damped)
   - Profile dropdown fades in, closes on click outside

7. TECHNICAL / BRUTALIST DETAILS:
   - Corner accents (2px border in corners of cards/modals) → signature look
   - Monospace IDs on every card (LOG-001, FARM-A102, STARTER_PLAN)
   - Code-comment style text in UI (e.g., "// Real-time monitoring")
   - All labels UPPERCASE with wide letter spacing (tracking-widest)
   - Pinging dots for live/active system status indicators
